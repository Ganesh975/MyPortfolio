HomeCare Services Web Application

Project Description: HomeCareServices is a dynamic web application developed using the Django framework, designed to streamline home service requests across cities. The platform connects service seekers with service providers by allowing users to request services such as plumbing and carpentry, while enabling service providers to showcase their profiles. This project aims to improve connectivity and facilitate efficient service management by bridging the gap between those in need of services and those who offer them.

Functionality

  • Farmer Role:
    • Registration/Login: Farmers can create personalized accounts with their contact details and login credentials.
    • Harvester Search: Location-based search functionality enables farmers to find suitable harvesters in their vicinity.
    • Harvester Selection: Farmers can view detailed profiles of harvesters, including machine details, pricing, and prerequisites to make informed choices.
    • Appointment Request: Farmers initiate service requests by providing essential details such as preferred date, land size, and required machine type.
    • Acknowledgment and Notification: Both farmers and harvesters receive notifications upon the acceptance of service requests.
    • Appointment Rejection Handling: Farmers are notified if their request is not acknowledged or rejected within 24 hours.
    • Service Completion: After the service is completed, farmers can provide feedback and ratings based on the harvester's performance.
  • Harvester Service Provider Role:
    • Registration/Login: Harvesters can register with detailed machine information, pricing, and prerequisites for personalized access.
    • Profile Management: Harvesters can manage their profiles by updating machine details and pricing.
    • Appointment Decision: Harvesters review and evaluate appointment requests based on their availability and farmer requirements.
    • Acknowledgment and Notification: Both farmers and harvesters receive notifications upon the acceptance of appointment requests.
    • Service Completion: Harvesters await feedback and ratings from farmers after service completion to enhance their service quality.
  • For Farmers:
    • Efficient Service Finding: Easily connect with suitable harvesters in the local area.
    • Time and Cost Savings: The streamlined process reduces the time spent searching for harvesters, optimizing agricultural activities.
    • Transparent Communication: The app ensures clear communication between both parties regarding service details.
    • Feedback-Driven Improvement: Farmers can provide valuable feedback to harvesters, contributing to service improvement in the agricultural community.

Technologies Used

  • Backend: Python (Django) - For developing the server-side logic and handling user requests.
  • Frontend: HTML, CSS, JavaScript - For creating a responsive and interactive user interface.
  • Database: SQLite3 - For storing user data and managing application data efficiently.
